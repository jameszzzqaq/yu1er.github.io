priority_queue自定义排序

10-7 · yuler | Edit
Table of Contents
  • 仿函数 (推荐)
  • lambda
  • 自定义类

给定一个pair<int, int>数组,通过自定义排序顺序决定优先队列中的顺序.

vector<pair<int, int>> nums{{4, 1}, {8, 2}, {1, 9}, {8, 1}, {8, 4}, {3, 7}};

仿函数 (推荐)#

仿函数的出现就是为了解决函数指针在作为函数参数向函数传递时的局限性问题. 通过重写类的operator(),使该类具有函数的功能.

struct cmp {
  bool operator() (const pair<int, int> &a, const pair<int, int> &b) {
    return a.first == b.first? a.second < b.second : a.first < b.first;
  }
};
priority_queue<pair<int, int>, vector<pair<int, int>>, cmp> pq;

lambda#

通过c++11新特性的lambda和delctype,可以避免重新定义类. 但需要注意,lambda函数需要通过构造函数的参数传递.

auto a = [](const pair<int, int> & a, const pair<int, int> &b){
  return a.first == b.first? a.second < b.second : a.first < b.first;
};
priority_queue<pair<int, int>, vector<pair<int, int>>, decltype(a)> pq(a);

自定义类#

默认对于自定义类的比较,priority_queue直接使用其operator<函数,因此如果需要比较,需要时先该运算符函数. 这种方法比较笨重,不适用自定义内置类的比较顺序.

struct node {
  int _a, _b;
  node(int a, int b) : _a{a}, _b{b}{};
  bool operator<(const node & r) const {
    return _a == r._a? _b < r._b: _a < r._a;
  }
};

vector<node> nums{{4, 1}, {8, 2}, {1, 9}, {8, 1}, {8, 4}, {3, 7}};
priority_queue<node> pq;
